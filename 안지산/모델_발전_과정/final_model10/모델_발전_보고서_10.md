### 2.2 Focal Loss 도입

#### 문제
```python
# Model 7~9: 일반 BCE Loss 사용
criterion = nn.BCELoss()
# → 쉬운 샘플과 어려운 샘플을 동등하게 취급
# → 어려운 경계 케이스 학습 부족
# → 클래스 불균형에 약함
```

#### 해결책 (Final Model 10)
```python
# ------------------------------------------------------------
# [변경점] final_model10: Focal Loss 적용
# ------------------------------------------------------------
class FocalLoss(nn.Module):
    """
    Focal Loss - 어려운 샘플에 집중하는 손실 함수
    
    수식: FL(p_t) = -α(1-p_t)^γ log(p_t)
    - p_t: 정답 클래스에 대한 예측 확률
    - γ (gamma): focusing 파라미터 (어려운 샘플 강조)
    - α (alpha): 클래스 불균형 가중치
    """
    def __init__(self, alpha=0.25, gamma=2.0):
        super().__init__()
        self.alpha = alpha
        self.gamma = gamma
    
    def forward(self, inputs, targets):
        # 1. Binary Cross Entropy
        BCE_loss = F.binary_cross_entropy(inputs, targets, reduction='none')
        
        # 2. p_t: 정답 클래스 예측 확률
        p_t = torch.where(targets == 1, inputs, 1 - inputs)
        
        # 3. Focal term: (1-p_t)^gamma
        focal_term = (1 - p_t) ** self.gamma
        
        # 4. Alpha weighting: 클래스 불균형 보정
        alpha_t = torch.where(targets == 1, self.alpha, 1 - self.alpha)
        
        # 5. 최종 Focal Loss
        loss = alpha_t * focal_term * BCE_loss
        return loss.mean()

# 사용
criterion = FocalLoss(alpha=0.25, gamma=2.0)
```

**개선 효과**:
- **어려운 샘플 집중**: 경계 케이스 학습 강화
- **클래스 불균형 대응**: alpha=0.25로 보정
- **성능 향상**: 일반화 능력 개선
- **안정적 학습**: 작은 변화에도 민감하게 반응

### 4.2 Focal Loss 효과 분석

#### 손실 함수 비교
```python
# BCE Loss
loss = -[y*log(p) + (1-y)*log(1-p)]

# Focal Loss
loss = -α(1-p_t)^γ [y*log(p) + (1-y)*log(1-p)]
      ↑   ↑      ↑
      │   │      └─ 기본 BCE Loss
      │   └──────── focal term (어려운 샘플 강조)
      └──────────── alpha (클래스 불균형 보정)
```

**작동 원리**:
1. **쉬운 샘플**: p_t ≈ 1 → (1-p_t)^γ ≈ 0 → 손실 작음
2. **어려운 샘플**: p_t ≈ 0.5 → (1-p_t)^γ 크음 → 손실 큼
3. **결과**: 모델이 어려운 경계 케이스에 집중 학습

**실제 효과**:
-  **Recall 향상**: 0.9828 → 0.9892 (+0.64%p)
-  **경계 케이스**: 모호한 샘플 학습 강화
-  **클래스 불균형**: alpha=0.25로 유해 샘플 보정

## 7. 기술적 핵심 발견

### 7.1 차원 축소의 위력


**교훈**:
 **"More is not always better"**
- 더 많은 차원이 항상 좋은 것은 아님
- 데이터 대비 적절한 차원 선택이 중요
- 차원 축소로 과적합 완화 가능

### 7.2 Focal Loss의 효과

```python
# Model 7~9: BCE Loss
loss = -[y*log(p) + (1-y)*log(1-p)]
# → 쉬운 샘플과 어려운 샘플 동일하게 처리
# → 경계 케이스 학습 부족

# Model 10: Focal Loss
loss = -α(1-p_t)^γ [y*log(p) + (1-y)*log(1-p)]
# → 어려운 샘플에 더 큰 가중치
# → 경계 케이스 집중 학습
```

**교훈**:
 **"Hard examples matter more"**
- 어려운 경계 케이스가 모델 성능의 핵심
- Focal Loss로 어려운 샘플에 집중
- 실제 배포 환경에서 더 강건한 모델

### 7.3 Early Stopping 기준의 중요성

```python
# Model 7~9: F1 기준 Early Stopping
if f1 > best_f1:
    save_model()
# → Threshold와 상관관계
# → 검증 세트에 과적합 위험

# Model 10: Val Loss 기준
if val_loss < best_val_loss:
    save_model()
# → Threshold와 독립적
# → 일반화 능력 향상
```

**교훈**:
 **"Choose the right metric"**
- F1는 Threshold와 상관관계
- Val Loss는 더 신뢰할 만한 일반화 지표
- Early Stopping 기준 선택이 중요
